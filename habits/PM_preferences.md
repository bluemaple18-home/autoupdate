## 🧠 核心行為鐵律

### 1. 非聖旨原則
- PM 看不懂 code，說的話不一定正確
- 有疑慮或方向不明，**立即提問**，不要埋頭做完才發現方向錯了
- 誠實原則：做錯了老實承認，一起解決，不粉飾太平

### 2. Karpathy Protocol（動扣前先分析）
- 接到任何需求，**必須先輸出 `👉 [假設與目標確認]`** 分析後，再開始產碼
- 禁止收到指令後直接撰寫程式碼或執行終端指令

### 3. 任務啟動協議
- 新任務或觸發技能時，必須先讀取並執行 `auto-skill` 的 `SKILL.md`
- 讀完才可開始作業

## 📋 卡片格式規格 v1（2026-04-15 決策）

每張任務卡（task.md 內）必須包含以下欄位：

| 欄位 | 說明 |
|------|------|
| **類型** | 實作卡 / 驗收卡 |
| **版本** | 語義版本號 |
| **卡號** | 英數代號（不得蓋過中文用途） |
| **用途** | 中文說明本卡目的 |
| **狀態** | 待開始 / 進行中 / NO-GO / GO |
| **範圍** | 本卡涵蓋的功能 |
| **不包含項目** | 明確排除在外的工作 |
| **證據路徑** | 驗收通過後截圖/log 的存放路徑 |
| **完成標準** | 清晰可量化的驗收條件 |
| **回報格式** | 回報時輸出的摘要結構 |
